cDock : Frequently Asked Questions

Getting Started

Read more by selecting the Documents dropdown in the sidebar and checking the links in the System Integrity Protection category.

You can also follow this link: What is SIP

It's likely that SIP wasn't properly disabled. You can open terminal and run the following command to see the current status of SIP on your machine.

csrutil status


The result should be:

System Integrity Protection status: disabled.

Try updating to the latest version of cDock.

If you're running cDock version 4.5 or earlier you can try adding the cDock Helper to your login items manually.  The helper process is located inside the main app at.

../cDock.app/Contents/Library/LoginItems/cDock Helper.app

Troubleshooting

When purchased directly from our website you should receive an email confirming the transaction and containing the license. Sometimes it may take a little longer, but at most 5-10 minutes.

When purchasing in-app your license should be automatically applied and can be viewed in the apps preferences. You should also receive an email containing your purchase and license info.

If you didn’t receive an email or lost your license the fastest way to retrieve it is from my.paddle.com — you might need to register first or use restore password form.

It’s likely you’re trying to use a cDock version 3 license with version 4.

cDock version 4 is a paid upgrade and is required for use on Big Sur. 

If you have not upgraded to Big Sur, you can continue to use cDock version 3 on Catalina and below. 

If you need a download link for an older version, or the current version you can find them here https://www.macenhance.com/docs/cdock/changes.html

First, make sure you’re using correct cDock version. cDock 3 and 4 versions use different licensing and are not compatible with each other.

If direct-download license is not activating, make sure you’re using correct email and not exceeding three activations per license, otherwise, deactivate it on other machines first.

If none of the above helps or after successful activation your copy of cDock stops working or asks for activation again please try restarting or reinstalling cDock. Make sure you have downloaded the latest version from our website depending on your license and try activating it again.

Most likely the cDock.bundle has not been loaded by the helper process. There are several reasons why this could happen, but most likely it has to do with System Integrity Protection being incorrectly configured.

You may also consider re-installing the app.

If you see the screen flashing black that means the Dock is repeatedly crashing. This is likely happening because you are using a version of cDock that is not compatible with your operating system.

The Dock changes to some degree with each new release of macOS so compatibility across older and newer macOS versions is not guaranteed.

If you are running a macOS beta and come across this behavior please get in touch so we can look into potential fixes.

You can just move cDock to the trash, empty the trash and restart.
To do a more through clean try and app like AppCleaner

Note that some settings simply control system Dock settings stored in
~/Library/Preferences/com.apple.dock.plist

These settings will stay as they were already set when cDock is deleted.
